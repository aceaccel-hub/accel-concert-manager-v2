아첼연주회관리프로그램 개발 기획서
==================

v0.1 / 1차 초안

작성일: 2026-06-08 문서 목적: 노션 회의록과 피드백을 기반으로 개발자가 바로 실행할 수 있는 요구사항 정리

이 문서의 핵심 방향
• 연주회 상세관리 화면을 중심으로 곡목, 단원, 연습, 예산을 연결합니다.
• 현재 확인된 저장 오류와 DB 연결 오류를 우선 수정합니다.
• 그린보트 미사 연주회를 기준으로 전체 흐름을 검증합니다.

문서 개요
=====

문서명 : 아첼연주회관리프로그램 개발 기획서
버전 : v0.1
작성일 : 2026-06-08
작성 목적 : 회의록과 피드백 내용을 바탕으로 개발 범위, 우선순위, 검수 기준을 실행형 문서로 정리
기준 시나리오 : 그린보트 미사 연주회 등록 및 상세관리 점검
주요 대상 : 공연 관리자, 운영 스태프, 개발자

1. 기획 배경
========

아첼연주회관리프로그램은 연주회 준비 과정에서 발생하는 공연 정보, 곡목, 작곡가, 단원, 파트, 연습 일정, 사례비, 수입/지출, 문서 자료를 한 곳에서 관리하기 위한 내부 운영 도구입니다.

현재 `그린보트 미사` 연주회 점검 과정에서 여러 저장 오류와 연결 오류가 확인되었으므로, 1차 개발의 핵심은 연주회 상세관리 화면을 중심으로 관련 데이터가 정확히 연결되고 유지되도록 하는 것입니다.

현재 확인된 문제
---------

- 연주회 단체 등록 후 상세관리에서 수정 흐름이 안정적으로 동작하는지 확인이 필요함
- 작곡가와 곡목 DB 연결, 기존 항목 선택, 신규 항목 입력 흐름에 오류가 있음
- 연습 일정 편집 시 시작/종료 시간, 대상 곡목 다중 선택, 예산/지출 기본 항목 표시가 부족함
- 수입/지출 저장 오류와 예상 금액 콤마 표시 오류가 있음
- 대시보드에서 그린보트 연주와 예산 정보가 지속적으로 노출되지 않는 문제가 있음
- 단원 관리에서 악기/파트 구분, 연락처, 계좌번호, 사례비 등 운영 필드 관리가 필요함

2. 제품 목표
========

2.1 핵심 목표
---------

- 연주회 상세관리 화면을 공연 운영의 중심 화면으로 정리한다.
- 곡목, 작곡가, 단원, 연습 일정, 예산, 대시보드 데이터를 연주회 기준으로 안정적으로 연결한다.
- 사용자가 입력한 수입/지출, 사례비, 연습 일정이 저장 후에도 사라지지 않도록 한다.
- `그린보트 미사` 연주회 등록부터 상세관리, 예산 확인까지 하나의 테스트 시나리오로 검증한다.

2.2 성공 기준
---------

- 사용자가 연주회 상세관리에서 기본 정보, 곡목, 단원, 연습, 예산을 수정한 뒤 새로고침해도 데이터가 유지된다.
- 기존 작곡가/곡목 선택과 신규 작곡가/곡목 추가가 모두 정상 동작한다.
- 연습 일정에 시작/종료 시간과 대상 곡목을 복수로 연결할 수 있다.
- 수입/지출 내역 저장이 정상 동작하고, 금액은 천 단위 콤마로 일관되게 표시된다.
- 대시보드에서 해당 연주회의 예산 정보가 사라지지 않고 계속 표시된다.

3. 주요 사용자
=========

3.1 공연 관리자
----------

- 연주회 정보를 등록하고 수정한다.
- 곡목, 작곡가, 단원, 파트, 사례비, 연습 일정을 관리한다.
- 수입/지출을 입력하고 예산 현황을 확인한다.

3.2 운영 스태프
----------

- 연습 일정과 단원 명단을 조회한다.
- 단원별 연락처, 계좌번호, 사례비 등 실무 정보를 확인한다.
- 문서 또는 단원 명단을 정리한다.

3.3 개발자/관리자
-----------

- 저장 오류, DB 연결 오류, 대시보드 연동 오류를 수정한다.
- 데이터 구조와 화면 흐름이 일관되게 작동하는지 검증한다.

4. 핵심 사용자 흐름
============

4.1 연주회 등록 및 상세관리
-----------------

1. `그린보트 미사` 연주회를 등록한다.
1. 등록 완료 후 연주회 상세관리 화면으로 이동한다.
1. 기본 정보, 곡목, 단원, 연습 일정, 예산 정보를 순차적으로 입력한다.
1. 각 영역 저장 후 새로고침 또는 재접속해도 데이터가 유지되는지 확인한다.
1. 대시보드에서 해당 연주회와 예산 정보가 정상 노출되는지 확인한다.

4.2 곡목/작곡가 관리
-------------

1. 상세관리에서 곡목 추가 버튼을 누른다.
1. 기존 곡목 또는 작곡가를 검색해 선택한다.
1. 기존 항목이 없으면 새 곡목 또는 새 작곡가를 입력한다.
1. 저장 후 연주회 곡목 목록에 연결 결과가 표시된다.
1. 이후 연습 일정에서 해당 곡목을 선택할 수 있다.

4.3 단원/파트/사례비 관리
----------------

1. 상세관리에서 단원을 추가한다.
1. 악기 또는 파트를 선택한다.
1. 연락처, 계좌번호, 기본 사례비를 입력한다.
1. 연주회별 사례비가 필요한 경우, 원본 단원 정보와 별도로 해당 연주회에서만 금액을 수정한다.
1. 저장 후 단원 목록과 예산 영역에 필요한 정보가 연결된다.

4.4 연습 일정 관리
------------

1. 상세관리 안에서 연습 일정을 추가한다.
1. 날짜, 시작 시간, 종료 시간, 장소를 입력한다.
1. 연습 대상 곡목을 복수 선택한다.
1. 필요 시 예산/지출 기본 항목을 확인한다.
1. 저장 후 연습 목록과 달력 보기에서 조회한다.

4.5 예산/수입/지출 관리
---------------

1. 상세관리에서 수입 항목을 추가한다.
1. `후원금`, `지원금` 등 기본 카테고리를 선택한다.
1. 예상 금액 또는 실제 금액을 입력한다.
1. 지출 항목도 동일한 방식으로 입력한다.
1. 저장 후 예산 요약과 대시보드에 반영된다.

5. 1차 개발 범위
===========

포함 범위
-----

- 연주회 상세관리 수정 흐름 안정화
- 작곡가/곡목 DB 연결 및 선택/신규 입력 오류 수정
- 단원 추가/편집 필드 정리
- 연주회별 사례비 오버라이드 구조 정리
- 연습 일정 입력 필드 보강
- 연습 일정과 곡목 다중 연결
- 수입/지출 저장 오류 수정
- 금액 콤마 표시 규칙 정리
- 대시보드 연주회/예산 정보 연동 상태 점검 및 수정

제외 범위
-----

- 외부 결제 연동
- 단원용 별도 로그인 화면

- 악보 파일 뷰어 고도화
- 외부 회계 시스템 연동

6. 화면별 상세 요구사항
==============

6.1 연주회 상세관리
------------

[목적]

연주회와 관련된 모든 운영 정보를 한 화면에서 관리하는 중심 화면입니다.

[필수 항목]

- 기본 정보
- 곡목/작곡가
- 단원/파트/사례비
- 연습 일정
- 예산/수입/지출
- 문서
- 대시보드 요약

[기능 요구사항]

- 등록된 연주회는 상세관리에서 수정 가능해야 한다.
- 각 섹션은 독립 저장이 가능하되, 저장 후 연주회 기준으로 연결되어야 한다.
- 저장 성공 시 사용자에게 성공 메시지를 보여준다.
- 저장 실패 시 어떤 필드에서 문제가 발생했는지 확인 가능한 오류 메시지를 보여준다.
- 새로고침 후에도 모든 입력값이 유지되어야 한다.

[검수 기준]

- `그린보트 미사` 연주회를 등록한 뒤 상세관리에서 수정 가능하다.
- 기본 정보 저장 후 목록, 상세, 대시보드에서 동일한 값이 보인다.
- 저장 실패 시 조용히 실패하지 않고 오류 안내가 표시된다.

6.2 작곡가/곡목 관리
-------------

[목적]

작곡가와 곡목을 DB 기반으로 관리하고, 연주회 및 연습 일정에 재사용할 수 있게 합니다.

[기능 요구사항]

- 기존 작곡가 검색 및 선택이 가능해야 한다.
- 기존 곡목 검색 및 선택이 가능해야 한다.
- 기존 항목이 없을 경우 신규 작곡가 또는 신규 곡목을 추가할 수 있어야 한다.
- 작곡가와 곡목은 관계형으로 연결되어야 한다.
- 동일한 작곡가/곡목이 중복 생성되지 않도록 최소한의 중복 확인이 필요하다.
- 곡목은 연주회에 연결된 뒤 연습 일정의 대상 곡목 선택 목록에 노출되어야 한다.

[검수 기준]

- 기존 작곡가 선택이 정상 동작한다.
- 기존 곡목 선택이 정상 동작한다.
- 신규 작곡가와 신규 곡목을 추가한 뒤 바로 선택할 수 있다.
- 선택한 곡목이 연주회 상세관리와 연습 일정 선택 목록에 모두 반영된다.

6.3 단원/파트/사례비 관리
----------------

[목적]

연주회 참여 단원과 파트, 연락처, 계좌번호, 사례비를 실무에 맞게 관리합니다.

[필수 항목]

- 이름
- 악기/파트
- 연락처
- 계좌번호
- 기본 사례비
- 연주회별 사례비
- 메모

[기능 요구사항]

- 단원 추가/편집 시 악기 또는 파트 선택이 가능해야 한다.
- 커서 이동과 탭 이동이 자연스럽게 동작해야 한다.
- 기본 사례비는 단원 원본 정보로 관리한다.
- 연주회별 사례비는 해당 연주회에서만 수정 가능한 별도 값으로 관리한다.
- 연주회에서 사례비를 수정해도 원본 단원 정보가 의도치 않게 변경되지 않아야 한다.
- 기본 사례비가 0인 단원도 등록 가능해야 한다.

[검수 기준]

- 파트 선택 후 저장하면 목록에 파트가 표시된다.
- 연락처, 계좌번호, 사례비가 저장 후 유지된다.
- 연주회별 사례비 수정 값이 해당 연주회 안에서만 반영된다.

6.4 연습 일정 관리
------------

[목적]

연습 일정을 연주회 상세관리 안에서 입력하고, 별도 메뉴에서는 보기 중심으로 활용합니다.

[필수 항목]

- 연습 날짜
- 시작 시간
- 종료 시간
- 장소
- 대상 곡목
- 참석 대상 또는 파트
- 메모

[기능 요구사항]

- 연습 일정 입력은 연주회 상세관리 안에서 처리한다.
- 별도 연습 메뉴는 목록/달력 보기 중심으로 정리한다.
- 시작 시간과 종료 시간을 명확히 입력할 수 있어야 한다.
- 대상 곡목은 연주회에 연결된 곡목 중에서 복수 선택할 수 있어야 한다.
- 파트별 악보 또는 파트 구성과 연결 가능하도록 데이터 구조를 고려한다.
- 저장 후 목록 보기와 달력 보기에서 일정이 표시되어야 한다.

[검수 기준]

- 연습 일정에 `몇 시부터 몇 시까지` 시간이 저장된다.
- 대상 곡목을 2개 이상 선택할 수 있다.
- 저장된 일정이 상세관리, 목록, 달력 보기에서 동일하게 표시된다.

6.5 예산/수입/지출 관리
---------------

[목적]

연주회별 수입과 지출을 입력하고 대시보드에서 예산 현황을 확인합니다.

[필수 항목]

- 수입 기본 카테고리: 후원금, 지원금, 티켓 수입, 기타 수입
- 지출 기본 카테고리: 출연료/사례비, 대관료, 악보/자료비, 홍보비, 운영비, 기타 지출

[기능 요구사항]

- 수입 항목 저장 오류를 수정한다.
- 지출 항목 저장 오류를 수정한다.
- 예상 금액과 실제 금액은 숫자 필드로 저장한다.
- 화면 표시 시 천 단위 콤마를 적용한다.
- 입력 중에는 사용자가 `2000000` 또는 `2,000,000` 형태 모두 자연스럽게 입력할 수 있게 한다.
- 저장 시 내부 값은 숫자로 정규화한다.
- 예산 요약은 수입 합계, 지출 합계, 잔액을 표시한다.

[검수 기준]

- 수입 항목을 저장하면 새로고침 후에도 유지된다.
- 지출 항목을 저장하면 새로고침 후에도 유지된다.
- `2000000` 입력 시 화면에는 `2,000,000`으로 표시된다.
- 대시보드의 예산 합계가 상세관리의 수입/지출 합계와 일치한다.

6.6 대시보드
--------

[목적]

현재 진행 중인 연주회와 예산 상태를 빠르게 확인하는 요약 화면입니다.

[기능 요구사항]

- `그린보트 미사` 연주회가 대시보드에 계속 표시되어야 한다.
- 해당 연주회의 예산 정보가 사라지지 않아야 한다.
- 연주회별 수입 합계, 지출 합계, 잔액이 표시되어야 한다.
- 데이터가 없을 때는 빈 화면 대신 `아직 등록된 예산 정보가 없습니다`와 같은 안내를 표시한다.
- 데이터 연결 오류가 있으면 관리자 또는 개발자가 확인할 수 있는 로그를 남긴다.

[검수 기준]

- 상세관리에서 예산을 입력하면 대시보드에 반영된다.
- 새로고침 후에도 예산 정보가 유지된다.
- 특정 연주회를 선택하거나 조회할 때 다른 연주회의 예산과 섞이지 않는다.

6.7 문서 관리
---------

[목적]

단원 명단, 연습 자료, 공연 운영 문서를 연주회 기준으로 정리합니다.

[기능 요구사항]

- 문서 영역은 우선 단원 명단 정리 용도로 사용한다.
- 연주회에 연결된 단원 목록을 문서화할 수 있도록 한다.
- 추후 악보, 계약서, 정산 자료 등 파일 관리 확장을 고려한다.

[검수 기준]

- 연주회 상세관리에서 단원 명단을 확인할 수 있다.
- 문서 영역에서 해당 연주회 기준의 자료임을 명확히 알 수 있다.

8. 우선순위
=======

P0: 즉시 수정 필요
------------

- 수입 저장 오류 수정
- 지출 저장 오류 수정
- 대시보드 예산 정보 사라짐 수정
- 작곡가/곡목 기존 항목 선택 오류 수정
- 연주회 상세관리 수정 데이터 유지 검증

P1: 1차 기능 완성
------------

- 연습 일정 시작/종료 시간 추가
- 연습 일정 대상 곡목 다중 선택
- 연습 일정 입력 위치를 연주회 상세관리 중심으로 정리
- 단원 파트 선택 및 탭/커서 이동 개선
- 연주회별 사례비 오버라이드 구조 적용

P2: 운영 편의 개선
------------

- 단원 명단 문서화
- 예산 카테고리 기본값 정리
- 문서 영역 확장
- 대시보드 빈 상태 안내 개선
- 악보/파일 관리 확장 검토

9. 개발 마일스톤 초안
=============

Phase 1. 저장 및 연결 오류 수정
----------------------

[목표]

현재 확인된 데이터 저장/연동 오류를 먼저 해결합니다.

[작업 범위]

- 수입/지출 저장 API 또는 상태 관리 로직 점검
- 금액 입력값 정규화 및 표시 포맷 수정
- 작곡가/곡목 선택 로직 점검
- 대시보드 예산 조회 로직 점검

[완료 기준]

`그린보트 미사` 기준으로 수입/지출 저장, 새로고침, 대시보드 반영이 정상 동작한다.

Phase 2. 상세관리 기능 완성
-------------------

[목표]

연주회 상세관리에서 운영 정보를 완결성 있게 입력할 수 있도록 합니다.

[작업 범위]

- 단원/파트/사례비 필드 정리
- 연주회별 사례비 구조 적용
- 연습 일정 시작/종료 시간 입력 추가
- 대상 곡목 다중 선택 적용

[완료 기준]

연주회 상세관리 안에서 곡목, 단원, 연습 일정, 예산 입력 흐름이 끊기지 않는다.

Phase 3. 조회/문서/대시보드 고도화
-----------------------

[목표]

운영자가 전체 상태를 빠르게 확인하고 문서화할 수 있도록 합니다.

[작업 범위]

- 연습 목록/달력 보기 정리
- 대시보드 요약 카드 정리
- 단원 명단 문서화
- 빈 상태 및 오류 메시지 개선

[완료 기준]

운영자가 대시보드와 문서 영역만 보고도 기본 운영 현황을 파악할 수 있다.

[ ] 새로고침 후 수정값이 유지된다.

11. 개발자 전달 메모
=============

- 연습 일정 입력의 기준 화면은 `연주회 상세관리`로 잡는 것이 좋습니다.
- 별도 연습 메뉴는 생성/편집보다 목록 및 달력 조회 중심으로 정리합니다.
- 사례비는 `단원 기본 사례비`와 `연주회별 사례비`를 분리해야 합니다.
- 수입/지출 금액은 화면 표시용 문자열과 DB 저장용 숫자를 분리해 처리해야 합니다.
- 대시보드는 연주회 ID 기준으로 예산 정보를 조회해야 하며, 조회 실패 또는 데이터 없음 상태를 명확히 구분해야 합니다.
- 작곡가/곡목은 기존 선택과 신규 생성이 같은 입력 컴포넌트에서 혼합될 가능성이 높으므로 중복 생성 방지와 선택 상태 초기화 로직을 함께 점검해야 합니다.

12. 결정 필요 사항
============

- 연주회 상세관리의 탭 구조를 확정해야 합니다.
- 곡목과 작곡가를 별도 DB로 둘지, 곡목 DB 안에 작곡가를 관계형으로 둘지 확정해야 합니다.
- 연습 메뉴를 완전히 보기 전용으로 둘지, 일부 빠른 편집을 허용할지 결정해야 합니다.
- 사례비 원본값과 연주회별 수정값의 우선순위를 확정해야 합니다.
- 예산 카테고리 기본값을 고정할지, 사용자가 추가 가능하게 할지 결정해야 합니다.

14. `그린보트 미사` 기준 통합 테스트 시나리오
============================

1. `그린보트 미사` 연주회를 등록한다.
1. 상세관리에서 기본 정보를 수정하고 저장한다.
1. 기존 작곡가와 곡목을 선택해 연주회에 연결한다.
1. 신규 작곡가와 신규 곡목을 추가해 연주회에 연결한다.
1. 단원을 추가하고 파트, 연락처, 계좌번호, 사례비를 입력한다.
1. 연주회별 사례비를 수정한다.
1. 연습 일정을 추가하고 시작/종료 시간과 대상 곡목을 복수로 선택한다.
1. 수입 항목에 `지원금 2,000,000원`을 입력하고 저장한다.
1. 지출 항목을 입력하고 저장한다.
1. 새로고침 후 모든 데이터가 유지되는지 확인한다.
1. 대시보드에서 `그린보트 미사`와 예산 정보가 정상 표시되는지 확인한다.

15. 최종 산출물 기준
=============

- 연주회 상세관리에서 공연 운영에 필요한 기본 데이터 입력이 가능하다.
- 입력한 데이터가 저장되고 새로고침 후에도 유지된다.
- 곡목, 단원, 연습, 예산이 연주회 기준으로 연결된다.
- 수입/지출 저장 오류가 해결된다.
- 대시보드에서 연주회와 예산 정보가 안정적으로 표시된다.
- `그린보트 미사` 테스트 시나리오를 통과한다.

개발 우선 판단
• 먼저 저장 오류와 대시보드 연동 오류를 해결해야 합니다.
• 그 다음 연습 일정과 단원/사례비 입력 흐름을 완성합니다.
• 마지막으로 문서 관리와 운영 편의 기능을 확장합니다.
